# 云点校园小程序埋点报表

## 数据表

| 表 | 用途 | 关键索引 |
| --- | --- | --- |
| `campus_analytics_event` | 保存启动、离开、页面浏览等有效行为事件 | 校区 + 事件 + 时间、校区 + 用户 + 时间 |
| `campus_analytics_session` | 保存前台会话和最后心跳，用于实时在线人数 | 校区 + 在线状态 + 最后活跃时间 |
| `campus_analytics_daily` | 预留每日归档，供数据量增长后的长期趋势查询 | 校区 + 日期唯一索引 |

事件属性只保存业务所需 JSON，不采集 OpenID、手机号等敏感身份字段；单次属性超过 8 KB 时会丢弃属性内容，但保留事件本身。

## 指标口径

- 注册用户：`campus_miniapp_user` 中未删除的用户；今日新增按 `create_time` 统计。
- 当前在线：`campus_analytics_session` 中仍在前台且 5 分钟内有心跳的去重登录用户。
- 今日活跃：今天至少产生一条有效埋点事件的去重登录用户。
- 今日发布：`campus_post` 今天创建且未删除的内容。
- 今日订单：`campus_trade_order` 今天创建且未删除的订单，包括待支付订单。
- 今日营收：今天支付、状态为已支付或已完成且未退款的订单金额之和。

## 采集与查询

- 小程序登录后通过 `POST /campus/analytics/track` 上报 `APP_SHOW`、`PAGE_VIEW`、`APP_HIDE`，前台每 60 秒发送一次 `HEARTBEAT`。
- 心跳只更新会话表，不写入事件明细，避免无效放大数据量。
- 管理后台通过 `GET /campus/analytics/overview` 查询全部校区或指定校区的 7、14、30 天实时趋势。
- 埋点请求失败会被小程序静默忽略，不影响登录、浏览、发布和交易主流程。
