# 云点校园项目快速了解

## 1. 项目一句话

云点校园是一个按校区独立运营的校园生活小程序，核心形态是：

```text
校园小红书 + 校园闲置交易 + 校区代理运营平台
```

它不是单纯的信息发布小程序，而是一个围绕“区域、校区、代理人、用户、内容、交易、分润”搭建的校园本地生活平台。

## 2. 项目要解决什么问题

### 对学生

- 快速找到本校区的二手闲置。
- 发布求助、失物招领、拼车、探店内容。
- 只看和自己校区相关的信息，减少无效内容。

### 对校区代理人

- 管理自己负责的校区。
- 推广用户、商家和内容。
- 后续通过订单、活动、商家入驻获得分润。

### 对平台

- 可以复制到多个区域和校区。
- 每个校区独立运营、独立统计。
- 后期支持分账、分润、裂变和商家商业化。

## 3. 最核心的设计

项目最重要的设计是：

```text
校区 = 租户
```

不要把校区只当成一个 `school_id` 字段。

每个校区都应该是一个独立租户，用户、内容、商品、订单、代理人、分润都围绕校区租户展开。

基本关系：

```text
平台
-> 区域
-> 校区租户
-> 校区代理人
-> 用户 / 内容 / 商品 / 订单 / 分润
```

这个设计是为了后续支持：

- 校区独立运营
- 校区数据隔离
- 校区代理人分润
- 邀请裂变
- 商家按校区入驻
- 多校区复制扩张

## 8. 下一步优先做什么

建议优先顺序：

1. 后台补齐区域、校区、代理人 CRUD。
2. 后端提供小程序校区列表接口。
3. 小程序把 mock 校区数据替换成真实接口。
4. 后端设计内容和商品表。
5. 后台生成内容和商品管理页面。
6. 小程序接入内容流和发布接口。
7. 接入微信登录和用户校区绑定。
8. 做订单基础流程。
9. 再做支付、分账、分润。

## 9. 不要一开始就做什么

当前不建议马上做：

- 复杂推荐算法
- 完整分账分润
- 多商家复杂结算
- 大规模消息系统
- 太细的 UI 动效
- 一次性重构底层框架包名

原因：

这些都很重要，但应该等基础业务跑通后再做。现在最重要的是先让一个校区能真实试点。

## 10. MVP 验收标准

第一阶段 MVP 做完，应该满足：

- 后台能创建区域。
- 后台能创建校区。
- 后台能给校区分配代理人。
- 小程序能选择校区。
- 小程序能浏览当前校区内容。
- 小程序能发布闲置或互助。
- 后台能审核内容和商品。
- 数据按校区隔离。
- 可以部署到服务器演示。
