# 云点校园微信小程序合规发布清单

> 适用版本：隐私政策 `2026-07-13`。本清单用于产品与技术验收，不替代运营主体的法律审核。

## 1. 微信公众平台必须配置

- 在「设置 → 服务内容声明 → 用户隐私保护指引」中补齐运营主体全称、注册地址、联系电话和可正常收件的隐私邮箱。
- 根据当前代码实际能力声明：
  - 用户信息（头像、昵称）：用于完善校园资料和社区身份展示；
  - 选中的照片或视频信息：用于头像和用户主动发布的内容图片；
  - 微信登录标识：用于识别账号、保持登录和保障账号安全；
  - 不声明、也不申请当前未使用的精确位置、通讯录、麦克风、蓝牙和后台定位。
- 每次新增权限、SDK、统计工具或信息处理目的时，先更新平台指引和小程序内隐私政策，再提审代码。
- 在体验版真机验证微信原生隐私授权弹窗、`chooseAvatar`、相册/相机选择和拒绝授权后的降级流程。

## 2. 当前代码已落实

- 游客启动时不调用 `uni.login`，不在用户未同意前创建后端账号。
- 登录页默认不勾选协议，用户可分别打开《用户协议》和《隐私政策》；拒绝后仍可浏览公开内容。
- `app.json` 开启 `__usePrivacyCheck__`，隐私政策按版本记录同意时间。
- 「设置与隐私」提供权限清单、本地数据清理、撤回同意、退出登录和账号注销。
- 注销接口会失效当前令牌，并删除或匿名化账号资料、发布、互动和举报关联信息。
- 内容详情提供真实举报接口；后台「举报处理」可查询、处理和记录结果。

## 3. 数据库与后台

- 部署新后端前执行 `campus-platform/sql/mysql/campus-content-governance-upgrade.sql`，创建举报表和后台菜单。
- 将菜单 `900900`、`900901`、`900902` 分配给校园运营角色；超级管理员默认可直接核验。
- 运营人员处理举报时只填写必要信息，不在处理说明中复制身份证号、手机号、住址等敏感内容。
- 设置安全日志留存与清理周期，限制数据库、对象存储和日志平台的访问权限。

## 4. 提审前人工核验

- 隐私政策中的运营主体、联系方式、域名和实际数据库/日志行为一致。
- `https://huanwoshidai.com.cn` 已配置 HTTPS，证书链有效，并加入微信 request/uploadFile/downloadFile 合法域名。
- 头像和内容图片的对象存储域名已加入合法域名；注销和删除内容后，按存储策略清理不再需要的原文件与备份。
- 发布页展示社区规范；详情页举报成功后，后台可以看到对应记录。
- 账号注销不要求新增人脸、身份证或其他未曾收集的信息，并在合理期限内完成处理。
- 未满十四周岁用户的使用限制和监护人同意规则已由运营主体确认。
- 运营人员具备投诉受理、侵权处理、紧急安全事件和执法协查流程。

## 5. 需要运营方最终确认的信息

- 运营主体法定全称；
- 注册地址与常用办公地址；
- 隐私负责人姓名或岗位；
- 客服电话、客服工作时间和隐私邮箱；
- 数据存储地点、日志保存期限和备份清理周期；
- 对象存储、短信、内容审核等第三方服务商清单及其处理信息范围。
