# 云点校园小程序开发计划

## 1. 项目目标

云点校园是一个按区域、校区独立运营的校园生活小程序，核心形态是“校园小红书 + 校园闲置交易 + 校区代理运营平台”。

第一阶段目标不是一次性做完整商业闭环，而是先做出可运营的 MVP：

- 后台能管理区域、校区、代理人、用户、内容、商品。
- 小程序能选择校区，浏览校园内容，发布闲置和互助信息。
- 所有业务数据都能按校区租户隔离，为后续分账、分润、裂变打基础。
- 后端、后台前端、小程序端都能持续自动部署和迭代。

## 4. 核心业务模块

### 4.1 区域与校区模块

业务意义：

区域和校区是整个平台的运营根基。后续代理人、分润、用户增长、内容推荐都围绕校区租户展开。

功能：

- 区域创建、编辑、停用。
- 校区创建、编辑、绑定区域。
- 校区绑定代理人。
- 校区配置邀请码、运营状态、服务范围。
- 小程序端选择当前校区。

关键数据：

- 区域
- 校区租户
- 校区资料
- 代理人
- 邀请码

## 6. 开发原则

- 先保证可运行，再追求完整。
- 先后台再小程序，后台数据结构稳定后，小程序接接口会更顺。
- 所有业务表必须带租户字段。
- 校区不是普通字段，校区就是租户。
- 订单、邀请、分润必须保留快照，避免后续规则变化导致历史数据错乱。
- 先用 mock 跑通页面，再替换真实接口。
- 每个阶段都要能部署、能演示、能回滚。
